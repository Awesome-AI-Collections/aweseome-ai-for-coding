# Anything Analyzer

## 一句话总结

一个把浏览器 CDP、MITM 抓包和 AI 分析串起来的桌面工具，可统一捕获网页、桌面应用、CLI、脚本和移动端流量，并自动生成协议逆向、安全审计和加密分析结果。

## 快速判断

- 如果你经常做网站 API 逆向、协议分析、签名排查或安全审计，它值得优先看
- 如果你只需要一个最基础的抓包代理，未必需要它这套 AI 分析和多源统一会话能力
- 它更像开发者直接使用的抓包与协议分析工作台，而不是单纯的 MCP 工具层

## 你会怎么用它

- 接进 AI 开发流：把浏览器、CLI、脚本、移动端流量统一抓进一个 session，再让 AI 做协议梳理、鉴权流程解释和加密分析
- 接进日常工作流：把原本分散在 DevTools、Charles/Fiddler、手工笔记之间的排查流程收拢到一个桌面工作台里
- 最小落地方式：先从一个明确目标开始，例如某个网页 API、某段 JS 加密流程或某个 CLI 请求链路，验证 AI 过滤和分析是否真能减少手工翻包时间

## 它解决什么问题

- 传统抓包工具往往只覆盖一类来源，浏览器、桌面应用、CLI、脚本和手机流量需要来回切工具
- 抓到大量请求后，真正费时间的通常不是捕获，而是筛选噪声、还原协议和理解签名逻辑
- 做安全审计或逆向时，经常需要把请求、存储、加密调用和多端行为放到同一个上下文里理解

## 适合谁

- 做网站 API 逆向、协议分析和 JS 加密排查的开发者
- 想把抓包、过滤和 AI 解读串成统一工作流的安全研究者
- 需要同时分析浏览器、CLI、脚本和移动端 HTTP 流量的工程人员

## 核心能力

- 多来源统一抓包：可同时覆盖网页、桌面应用、终端命令、脚本程序和手机代理流量
- 双通道捕获：结合浏览器 CDP 与 MITM 代理，把不同来源汇总进统一 session
- AI 两阶段分析：先过滤噪声请求，再聚焦做协议、鉴权、安全或加密分析
- JS Hook 注入：可拦截 `fetch`、XHR、`crypto.subtle`、CryptoJS、SM2/3/4 等相关调用
- MCP 集成：既能接外部 MCP Server，也能把自身抓包分析能力暴露给 Claude Desktop、Cursor 等工具

## 能力边界

- 明显可用：HTTP/HTTPS 抓包、协议梳理、签名与加密排查、安全审计、逆向文档整理
- 效果一般：如果目标不是网络协议问题，而是纯代码逻辑 bug，它的价值会下降
- 不要误用：它不是通用漏洞利用框架，也不是“任何应用都能无障碍完全逆向”的保证工具
- 文档里明确写了只读捕获、不修改请求响应，WebSocket 仅隧道转发且不做解密，单个 body 还有大小限制

## 集成方式

- 作为单独工具：直接用内嵌浏览器和 MITM 代理抓流量，再做 AI 分析
- 作为 AI 工作流组件：放在协议采集与预分析层，把抓到的流量和解释结果喂给后续 agent 或审计流程
- 作为团队流程节点：适合在逆向、安全评估和接口排查阶段减少手工筛包与解释成本

## 上手建议

- 第一轮先只抓一个来源，不要同时把浏览器、手机和脚本全开上来
- 最值得先试的是目标明确的单条链路，例如登录流程、签名接口或 CLI 请求序列
- 如果你的核心目标是广义网络调试而不是 AI 分析，先轻量试用，确认它的 AI 分析质量是否匹配你的需求

## 选型建议

- 如果你的主需求是“抓到流量后快速理解协议和安全风险”，它很有吸引力
- 如果你的主需求只是稳定代理抓包，传统抓包工具可能已经够用
- 如果你更在意把抓包结果进一步接进 Claude Desktop、Cursor 等 agent 工作流，它的 MCP 设计值得看

## 典型使用场景

- 逆向网站 API，输出接口说明、鉴权链路和复现思路
- 分析 App 或 H5 的签名、加密和请求流转
- 调试 CLI 工具、脚本或微服务之间的 HTTP 调用链路
- 对浏览器和代理混合流量做安全审计与敏感数据排查

## 为什么值得关注

- 它抓住了真实痛点：抓包不是最难的，真正难的是跨来源整合和后续解释
- 这类工具对 AI 最自然的落点不是“帮你聊天”，而是“帮你筛、帮你读、帮你还原”
- MCP 支持让它不只是一个桌面工具，也有机会成为更大 agent 工作流里的协议分析节点

## 类似项目

- [METATRON](metatron.md) - 同样把 AI 和安全分析结合起来，但 METATRON 更偏本地渗透测试工具链；Anything Analyzer 更聚焦流量抓取、协议逆向和请求分析
- [Cloak Browser](cloak-browser.md) - 同样与浏览器自动化和网页行为有关，但 Cloak Browser 更偏自动化与隐私隔离；Anything Analyzer 更偏抓包与协议理解
